📖 How To Use This Guide ▶

This is both a process manual and a training reference for the Metalmark weekly Cash and Unpaid Bills Report. It was built from the recorded training session on May 19, 2026, and is designed to support you whether you're learning the process for the first time or looking up a specific step mid-run.

🎬
Learning the process for the first time
Watch Videos 1–4 in order. Use the video cards above to open each one. Then run through the process yourself — refer back to the video sections in this guide if you get stuck on a specific step.
⌕
Stuck on a specific step mid-run
Use the search bar at the top. Type any keyword (e.g. "refund," "reconcile," "pending," "journal entry") and results will show you the step description plus the exact video timestamp to jump to.
✓
Running the report each week
Open the Run Checklist (left sidebar) and tick off each step as you complete it. Covers all three videos in order. Reset when done.
🏪
Unsure which company to use
Open the Three Companies quick reference (left sidebar) for a summary of Metalmark, Carmel, and Phoenix — their roles, merchant accounts, and common quirks.
ℹ
Process flexibility is by design. As Garrett noted in the training: "As long as accurate cash and bill data reaches Mary, the exact steps can evolve." This manual documents the current established process — if you and Mary agree to streamline any steps (e.g. skip the Previous Day comparison column, or skip combining PDFs), update the checklist accordingly.

Credit Card Batches

Pull batch totals from M Reports, match to Chase deposits, enter QBO journal entries for each batch.

Video 2 · ~33 min
▶ Watch on Vimeo
📋 Step 1 — Set Up the Excel Spreadsheet for Today V2 ~0:28 ▶
  • 1
    Open the month's Excel file
    Navigate to Accounting → Year → Month folder. One spreadsheet per month to avoid hundreds of tabs.
    ~0:51
  • 2
    Duplicate the last date's tab
    Right-click the most recent date tab → Move or Copy → check "Create a copy." Rename it to today's date (e.g., "19").
    ~1:19
  • 3
    Paste Previous Day values
    Select the entire "Current Day" column → Ctrl+C → click the "Previous Day" column → right-click → Paste Special → Values only. This freezes yesterday's numbers without breaking formulas.
    ~2:15
  • 4
    Zero out all Current Day fields
    Clear all values in the Current Day column — you'll refill them with today's data in the steps below.
    ~3:04
✔
The spreadsheet compares Current Day vs Previous Day. If Mary only wants current balances, the Previous Day step can be skipped to save time.
💳 Step 2 — Get Batch Totals from M Reports V2 ~5:33 ▶
  • 1
    Log into M Reports for Phoenix (account #1863)
    Enter credentials → MFA code sent to Gmail → copy code → log in → delete email immediately.
    ~5:53
  • 2
    Run Batch Summary report
    Go to Reports → Batch Summary. Set date range back ~1 week to capture all recent batches. Note: some reports cap at 7 days.
    ~7:27
  • 3
    Copy batch totals into Excel (Phoenix section)
    Copy & paste each batch date and total sales amount. Each day gets its own line item — if multiple batches on the same day, add separate rows.
    ~7:59
  • 4
    Log out, then log into Carmel (account #1897)
    Repeat the Batch Summary process for Carmel. A new MFA code will be sent to Gmail each time you log in.
    ~9:41
  • 5
    Watch for negative batches (returns)
    A return creates a negative batch. Enter the return amount as a negative number in Excel. On the same date there will typically be two separate bank transactions: the return going OUT and the regular deposit going IN. Each gets its own row.
    ~11:00
⚠
Date consistency matters. The date you enter in Excel, the date on M Reports, and the journal entry date in QBO must all match for reconciliation to work cleanly. (V2 ~12:11)
🏦 Step 3 — Match Deposit Amounts in Chase V2 ~16:58 ▶
  • 1
    Open Chase for Carmel, then Phoenix
    Look for deposits described as "Merchant Services" or "Aurora." Pending items (today's) show with a pending flag and won't download to QBO until tomorrow.
    ~16:58
  • 2
    Enter actual deposit amounts in Excel
    The deposit amount is slightly less than the batch total — the difference is the merchant fee. The spreadsheet formula auto-calculates the fee % for you.
    ~15:01
  • 3
    Verify merchant fee % is between 2%–3%
    The spreadsheet calculates (Batch Total − Deposit) ÷ Batch Total. It should come out to 2%–3%. If it's outside that range, you probably matched the wrong batch to the wrong deposit. Re-check your pairing.
    ~15:01
  • 4
    Handle return transactions in Chase
    A return day will show TWO separate items in Chase: one debit (return going out, e.g. −$2,628.45) and one credit (net positive batch deposited, e.g. +$3,228.87). Enter both as separate rows in Excel.
    ~13:01
✔
Pending transactions: Even if a transaction is still "pending" in Chase (not yet cleared), include it in the report as a line item. This is how you catch material amounts that haven't downloaded to QBO yet. (V2 ~22:27)
📒 Step 4 — Enter QBO Journal Entries (Per Batch) V2 ~20:36 ▶

Each credit card batch gets its own journal entry. Do Phoenix first, then Carmel. Use the most recent JE as a template by copying it forward.

  • 1
    Open QBO for Phoenix → + → Journal Entry
    Click the clock/history icon to see past journal entries. Open the most recent one (highest number / latest date).
    ~23:22
  • 2
    Copy it: More → Copy
    Critical: always copy, never overwrite. Overwriting changes historical bookkeeping. The copy will say "This is a copy" in the memo — that's your safety signal.
    ~26:02
  • 3
    Update the date and amounts
    Change the JE date to match the batch date. Fill in: Credit → Sales (batch total), Debit → Accounts Receivable (deposit amount to Chase), Debit → Merchant Card Expense (the fee).
    ~26:05
  • 4
    Save → then Copy again for the next batch date
    Save the current entry, then immediately go to More → Copy to create the template for the next date. Repeat for each batch date until all are entered.
    ~25:37
  • 5
    Switch to Carmel and repeat
    Same process. The last JE you use for regular batches becomes your template — do refunds separately using the 14th's entry as a base.
    ~28:15
⚠
Refund Journal Entries are different. Instead of Credit → Sales, it's Debit → Sales (negative sale) and Credit → Accounts Receivable. There is NO merchant fee on a refund. See Video 3 for matching the refund in Chase. (V2 ~32:03)
ℹ
The vendor name in QBO for Aurora deposits is "Chosen" (a legacy name from when Aurora was called Aurora & Chosen). Don't update it — QBO doesn't handle vendor renames gracefully.
03

Update QBO Transactions & Reconcile

Refresh bank feeds, match and categorize all downloaded transactions, then reconcile all three accounts.

Video 3 · ~28 min
▶ Watch on Vimeo
🔄 Step 1 — Update Bank Feeds in All Three Companies V3 ~0:04 ▶
  • 1
    In each QBO company: Accounting → Bank Transactions → Update
    Click the Update button to pull in all new transactions from the bank. Do this for Carmel, Phoenix, and Metalmark before you start matching anything.
    ~0:47
ℹ
Transactions posted today at the bank download into QBO the next business day. That's why pending transactions are tracked manually in the report — QBO won't have them yet.
✅ Step 2 — Categorize & Match Transactions V3 ~1:20 ▶

For each downloaded transaction you have two options: Match it to an existing ledger entry, or Post it to a new ledger account. Always be 100% sure before posting.

  • 1
    Credit card deposits → Match to Journal Entries
    QBO should auto-match deposits to the A/R entries you created in Video 2. If not, click "Match" manually. The match creates a payment receipt that reduces A/R and increases Cash.
    ~2:06
  • 2
    Handle unrecognized expenses
    If you can identify it (Jimmy John's → Meals & Entertainment, FedEx → Shipping), post it. If unsure, post to Uncategorized Expenses and review with Mary later. Don't post something to the wrong account just to clear it.
    ~2:55
  • 3
    Amex transactions — do credit card side FIRST
    Amex payments download twice: once from Amex, once from Chase. Always match the Amex-side first, then the Chase side will auto-match. If you do it in the wrong order you can create a duplicate payment.
    ~11:35
  • 4
    Refund transactions — two-step process
    1. Find the return in Chase (it's a debit going out). It won't auto-match.
    2. Create an Expense to "Chosen" posted to Accounts Receivable for the return amount.
    3. Go to Receive Payment → Customer "Chosen" → match the journal entry credit side against this new expense.
    This closes out the outstanding A/R balance.
    ~6:06
  • 5
    Transfers between entities
    When Metalmark transfers funds to Carmel/Phoenix (or vice versa), it downloads on both sides. Post as a Transfer or A/P on the sending side and A/R on the receiving side. Make sure both sides get entered.
    ~10:16
⚠
Downloaded ≠ Posted. A transaction appearing in QBO's bank feed is NOT in your books yet. You must either Post it or Match it. Non-accountants often miss this distinction and accidentally post duplicate entries. (V3 ~22:34)
⚖️ Step 3 — Reconcile All Three Accounts V3 ~15:56 ▶
  • 1
    For each company: open Reconciliation → Resume Reconciling
    Go to Accounting → Reconcile → select the Chase account → Resume Reconciling (not Start New — you use ongoing reconciliation).
    ~15:56
  • 2
    Edit Info → update Statement Ending Balance
    Enter the last available balance from Chase (as of the most recent business day). The difference should be $0 if everything was entered correctly.
    ~16:27
  • 3
    Review all outstanding items — even if difference = $0
    A duplicate entry can still leave you reconciled at $0 if the debit and credit cancel out. Always scroll through outstanding checks and bills to verify they're legitimate and not old duplicates.
    ~17:18
  • 4
    Save for Later — do NOT click Finish
    Clicking Finish posts a formal reconciliation in QBO. Use "Save for Later" so you can continue incrementally. Only Finish at month-end.
    ~17:57
⚑
Why reconcile even when balanced? You could have a $10,000 duplicate check in outstanding items that's offsetting a real deposit — you'd be reconciled but under-reporting cash by $10K. The review catches that. (V3 ~17:18)

🛠 Troubleshooting: Duplicate Rent / Bill Issues V3 ~19:02 ▶

A common issue: a recurring bill is set at an old amount. When the ACH clears at a different amount, both the old and new entries are in QBO.

  • 1
    Find what actually cleared Chase
    Search Chase for the vendor name and see the actual amount that debited.
  • 2
    Delete the bill payment in QBO (not the bill itself)
    Delete the erroneous bill payment, then match the downloaded transaction to the correct bill. Update the bill amount to the actual paid amount.
    ~21:27
  • 3
    Update the recurring template amount
    If rent changes periodically, update the recurring transaction template so it defaults to the new amount going forward.
    ~26:16
04

Populate Report & Send Email

Enter reconciled ledger balances into Excel, run unpaid bills PDFs, combine them, and email to Mary.

Video 4 · ~15 min
▶ Watch on Vimeo
📊 Step 1 — Fill Excel Report with Ledger Balances V4 ~0:04 ▶
  • 1
    Use the QBO Ledger Balance — not the bank balance
    The ledger balance accounts for outstanding checks and bill payments that have been entered but not yet cleared. Bank balance is always different.
    ~0:04
  • 2
    Enter balances for each company
    Carmel: QBO Chase ledger balance
    Phoenix: QBO Chase ledger balance
    Metalmark: QBO Chase ledger balance + Amex balance
    ~0:26
  • 3
    Enter Shopify pending amount
    Check Shopify for any sales that have not yet been transferred to Chase. These are "in transit" funds. Enter the pending payout total as a line item. If everything has already paid out, enter $0.
    ~1:51
  • 4
    Enter any pending credit card amounts
    From Video 2 Step 3 — any M Reports batches that are still "pending" in Chase (not yet deposited) are entered here as Credit Cards In Transit. This captures money earned but not yet in the bank.
    ~2:43
⚑
When Mary asks "how much cash do we have to pay bills?", the answer is: Total Ledger Cash + Pending Credit Card Receivables. The report shows both.
🧾 Step 2 — Run Unpaid Bills Reports from QBO V4 ~3:07 ▶
  • 1
    Metalmark — use Standard Unpaid Bills report
    Reports → Standard → Unpaid Bills. Scan for any weird negative amounts or anything that looks like a duplicate. Note: the regular Carmel report may show confusing internal Metalmark entries — use the custom "Unpaid Bills Report Daily" for Carmel instead.
    ~3:07
  • 2
    Carmel — use custom "Unpaid Bills Report Daily"
    The standard Carmel report shows confusing Metalmark vendor entries. The custom report filters those out for a cleaner view.
    ~10:08
  • 3
    Phoenix — use Standard Unpaid Bills report
    Same as Metalmark. Export to PDF.
    ~11:16
  • 4
    Export each to PDF → save to Accounting/Month/Unpaid Bills folder
    File → Export → PDF. Name them clearly (e.g., Unpaid_Bills_0519_Metalmark.pdf, etc.) so you can combine them in the next step.
    ~8:39
✔
Spot-check bills before exporting. Look for duplicate invoices (same vendor, same amount, entered twice), old stale bills from prior years, and any negative bills that may indicate unmatched payments. (V4 ~3:49)
📎 Step 3 — Combine PDFs in Adobe Acrobat V4 ~11:26 ▶
  • 1
    Open Metalmark PDF in Acrobat
    This is your base document.
  • 2
    Right-click in Pages panel → Insert Pages from File
    Insert Carmel PDF before the end, then insert Phoenix PDF. Keep order consistent every time (Metalmark → Carmel → Phoenix).
    ~11:38
  • 3
    Save the combined file
    Save as the final Unpaid Bills PDF (e.g., Unpaid_Bills_0519_Combined.pdf).
ℹ
If you don't have Acrobat, you can send Mary three separate PDFs — it just takes longer for her to review. Adobe Acrobat Basic is ~$12–15/month.
📧 Step 4 — Compose and Send Email to Mary V4 ~12:25 ▶
  • 1
    Create new email: To → Mary, CC → Alex
    Subject line: Metalmark Cash and Unpaid Bills
    ~12:25
  • 2
    Paste the Excel report as an image
    In Excel, select all cells of the report → Ctrl+C → in the email, paste as Image (not as a table). This renders better on mobile for Mary.
    ~13:05
  • 3
    Attach the combined Unpaid Bills PDF
    Double-click the file to confirm it's the right one before attaching.
    ~13:24
  • 4
    Note any pending items in the email body
    If any credit card deposits are still "pending" (not yet received at the bank), mention it: e.g., "The $995.28 Carmel deposit is still pending as of today."
    ~2:43
  • 5
    Double-check everything and hit Send
    Verify the unpaid bills total in the email matches what's in the spreadsheet, then send.
    ~14:12
